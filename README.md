# Spotify Clone 🎵

A sleek, responsive music player web application inspired by Spotify. This project demonstrates the power of vanilla JavaScript, DOM manipulation, and the HTML5 Audio API to create a functional and visually appealing user interface.

## 🚀 Why This Project?

This project was created to:
- **Master DOM Manipulation**: Dynamically updating song lists, titles, and cover art based on user interaction.
- **Understand Audio API**: Implementing play, pause, seek, and track navigation functionality.
- **Practice Responsive UI**: Designing a layout that looks great across different screen sizes using sticky positioning and flexible containers.
- **Learn Event Handling**: Managing complex user interactions like progress bar updates and track switching.

## ✨ Features

- **Dynamic Song List**: Automatically populates the UI with songs, their covers, and durations defined in the JavaScript.
- **Custom Music Player**:
    - **Play/Pause**: Seamlessly control music playback.
    - **Track Navigation**: Quickly switch between tracks with 'Next' and 'Previous' buttons.
    - **Interactive Progress Bar**: A custom seek bar that updates in real-time as the song plays and allows users to jump to any part of the track.
- **Visual Feedback**:
    - Animated "playing" GIF that syncs with the music playback.
    - Hover effects and cursor pointers for better interactivity.
- **Modern Typography**: Uses Google Fonts (Ubuntu and Varela Round) for a premium feel.

## 🛠️ Tech Stack

- **HTML5**: For semantic structure.
- **CSS3**: For advanced styling, layout, and responsiveness.
- **JavaScript (Vanilla)**: For core logic and interactivity.
- **Font Awesome**: For high-quality UI icons.
- **Google Fonts**: For elegant typography.

## 📂 Project Structure

- `index.html`: The main structural file.
- `style.css`: Contains all visual styles and layout definitions.
- `app.js`: The heart of the application, managing audio and UI state.
- `songs/`: Audio files for the tracks.
- `covers/`: Album art for each track.
